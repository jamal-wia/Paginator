## What it is

**Interweaving** is an opt-in transformation over a paginator's
`Flow<PaginatorUiState<T>>` that inserts meta-rows — *decorations* — between
data items. Typical decorations are date headers ("Today", "Yesterday"),
alphabetical section headers, "unread since" dividers, inline banners, etc.

The feature is implemented as **a single `Flow` operator** that lives next to
the paginator but does not touch it: the paginator core, cache, CRUD,
serialization, DSL and `PaginatorUiState` are completely unchanged. If you do
not call `.interweave(...)`, nothing changes for you.

## When to use it

- Your list needs section headers or dividers derived purely from the current
  set of visible items (dates, buckets, unread boundary, …).
- You want a flat `List<WovenEntry<T, I>>` suitable for a `RecyclerView`
  adapter or a Compose `items { }` block.

When your headers are truly server-driven (the backend returns the header as
part of the page payload), skip this feature — model the header as part of a
sealed item type at the data layer and paginate that sealed type directly.

## API surface

Three public symbols, one optional builder:

```kotlin
// The sealed output row type.
sealed interface WovenEntry<out T, out I> {
    val wovenKey: Any

    data class Data<T>(val value: T, override val wovenKey: Any) : WovenEntry<T, Nothing>
    data class Decoration<I>(val value: I, override val wovenKey: Any) : WovenEntry<Nothing, I>
}

// The strategy.
interface Interweaver<T, I> {
    fun itemKey(item: T): Any
    fun decorationKey(decoration: I, prev: T?, next: T?): Any
    fun between(prev: T?, next: T?): I?
    val emitLeading: Boolean get() = false
    val emitTrailing: Boolean get() = false
}

// The Flow operator (the only integration point).
fun <T, I> Flow<PaginatorUiState<T>>.interweave(
    weaver: Interweaver<T, I>,
): Flow<PaginatorUiState<WovenEntry<T, I>>>

// Optional DSL builder.
inline fun <T, I> interweaver(block: InterweaverBuilder<T, I>.() -> Unit): Interweaver<T, I>
```

There is also a direct `PaginatorUiState<T>.interweave(weaver)` overload (for
places that hold a single snapshot) and a public `List<T>.weave(weaver)` helper
(for callers who manage items outside of `uiState`).

## Basic example — date headers

```kotlin
data class DateLabel(val date: LocalDate)

val dateWeaver = interweaver<Message, DateLabel> {
    itemKey { it.id }
    decorationKey { label, _, _ -> "date-${label.date}" }
    between { prev, next ->
        val prevDate = prev?.sentAt?.toLocalDate()
        val nextDate = next?.sentAt?.toLocalDate()
        if (prevDate != nextDate && nextDate != null) DateLabel(nextDate) else null
    }
}

paginator.uiState
    .interweave(dateWeaver)
    .onEach(::render)
    .launchIn(viewModelScope)
```

`between(prev, next)` is consulted at every adjacent pair and returns either
the decoration to insert or `null` to insert nothing. Because the operator sees
the already-flattened `items` list from `PaginatorUiState.Content`, decorations
never duplicate at page seams — a single "Today" header is emitted for a run
of same-date messages spanning multiple loaded pages.

## How boundary states are handled

`PaginatorUiState.Idle`, `Loading`, `Empty` and `Error` do not carry item data
and pass through structurally unchanged.

For `PaginatorUiState.Content`:

- `items` is transformed to `List<WovenEntry<T, I>>` via the weaver.
- `prependState` / `appendState` (the non-success boundary indicators) have
  every element of their `data` wrapped in `WovenEntry.Data`. This keeps the
  indicator's previously-visible items available to the UI with the same row
  dispatch logic used for regular content.

## Recipe — unread divider

A one-shot divider between the last read and first unread message:

```kotlin
object UnreadLabel

val unreadWeaver = interweaver<Message, UnreadLabel> {
    itemKey { it.id }
    decorationKey { _, _, _ -> "unread-divider" }
    between { prev, next ->
        if (prev?.isRead == true && next?.isRead == false) UnreadLabel else null
    }
}
```

## Rendering

In a Compose `LazyColumn`:

```kotlin
LazyColumn {
    items(content.items, key = { it.wovenKey }) { entry ->
        when (entry) {
            is WovenEntry.Data<Message> -> MessageRow(entry.value)
            is WovenEntry.Decoration<DateLabel> -> DateHeader(entry.value)
        }
    }
}
```

In a `RecyclerView` adapter, dispatch `getItemViewType` on the sealed variant
and use `wovenKey` for `DiffUtil` item identity.

## What it does not touch

- `Paginator` / `MutablePaginator` — no new properties, no generic changes.
- `PagingCore`, caches, eviction — decorations never enter the cache.
- CRUD (`addElement`, `removeElement`, `setElement`, …) — operates on `T`.
- Serialization — decorations are re-derived from the current snapshot on
  every emission, so persisted state is identical with or without a weaver.
- The DSL builder (`paginator { }` / `mutablePaginator { }`) — the weaver is
  attached at the `Flow` call site, not in the builder.
- `PaginatorUiState` — still a single-generic sealed interface. Interweaving
  reparameterises it from `T` to `WovenEntry<T, I>`; it does not add a new
  variant.
