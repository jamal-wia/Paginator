## File layout

All extensions live in `com.jamal_aliev.paginator.extension` and are organised by topic:

| File                      | Receiver           | What's inside                                                              |
|---------------------------|--------------------|----------------------------------------------------------------------------|
| `PageExt`                 | `PageState`        | `is*State` predicates, `near`/`far`/`gap`                                  |
| `PaginatorIterationExt`   | `Paginator`        | `forEach`, `smartForEach`, `walkForwardWhile`, `walkBackwardWhile`         |
| `PaginatorAggregationExt` | `Paginator`        | `find`, `count`, `flatten`, `firstOrNull`, `contains`, ‚Ä¶                   |
| `PaginatorCrudExt`        | `MutablePaginator` | `addElement`, `prependElement`, `moveElement`, `removeAll`, `updateAll`, ‚Ä¶ |
| `PaginatorRefreshExt`     | `Paginator`        | `refreshAll`                                                               |
| `PaginatorPrefetchExt`    | `Paginator`        | `prefetchController`                                                       |
| `PaginatorUiStateExt`     | `Paginator` / Flow | `uiState` property, `asUiState`, `toUiState`                               |
| `PaginatorPersistentExt`  | `Paginator`        | `warmUpFromPersistent` ‚Äî eager L1 restore from L2 on cold start            |

The package is the same for every file, so existing imports
(`com.jamal_aliev.paginator.extension.smartForEach`, etc.) keep working without changes.

## `PageExt` ‚Äî PageState predicates and distance

Type-checking with Kotlin contracts (enables smart-casting after the check returns `true`):

```kotlin
pageState.isProgressState()   // true if ProgressPage
pageState.isSuccessState()    // true if SuccessPage (but NOT EmptyPage)
pageState.isEmptyState()      // true if EmptyPage
pageState.isErrorState()      // true if ErrorPage

// Specific custom subclasses
pageState.isRealProgressState(MyCustomProgress::class)
pageState.isRealSuccessState(MyCustomSuccess::class)
pageState.isRealEmptyState(MyCustomEmpty::class)
pageState.isRealErrorState(MyCustomError::class)
```

Distance helpers (infix):

```kotlin
pageA near pageB   // gap is 0 or 1
pageA far  pageB   // gap > 1
pageA gap  pageB   // |a.page - b.page|
```

## `PaginatorIterationExt` ‚Äî iteration & traversal

```kotlin
// All cached PageStates in ascending page order
paginator.forEach { pageState -> /* ‚Ä¶ */ }

// Customised iteration: choose initial index, step, and stop condition
paginator.smartForEach(
    initialIndex = { states -> 0 },
    step = { it + 1 },
) { states, index, currentState ->
    /* return true to keep going, false to stop */
    true
}

// Walk through contiguous pages starting from a pivot
paginator.walkForwardWhile(pivotState)  { it.isSuccessState() }
paginator.walkBackwardWhile(pivotState) { it.isSuccessState() }
```

`smartForEach` is the underlying primitive used by the navigation code itself; expose it when
you need full control over the iteration order. `forEach` is a thin convenience wrapper.

## `PaginatorAggregationExt` ‚Äî search, aggregation, flat access

This file groups everything that *reads* across multiple cached pages ‚Äî searches, predicates,
flat-list access, and reduction.

### Search

```kotlin
// First / last element matching a predicate (across all pages, page order)
val article = paginator.find { it.id == targetId }
val article = paginator.findLast { it.draft }

// Pair<Int, Int>? ‚Üí (page, index) of the first/last match
val coords = paginator.indexOfFirst { it.id == targetId }
val coords = paginator.indexOfLast  { it.draft }

// Same, but constrained to a single page
val coords = paginator.indexOfFirst(page = 3) { it.id == targetId }
val coords = paginator.indexOfLast (page = 3) { it.draft }

// Historical alias kept for back-compat ‚Äî prefer `find` in new code
val article = paginator.getElement { it.id == targetId }
```

### Predicates

```kotlin
paginator.any  { it.unread }                    // true if at least one element matches
paginator.none { it.unread }                    // true if no elements match
paginator.all  { it.publishedAt > yesterday }   // true if every element matches (vacuously true on empty)

paginator.count()                               // total loaded items across all pages
paginator.count { it.unread }                   // matching elements only
```

### Positional access

```kotlin
paginator.firstOrNull()                 // first loaded element, or null
paginator.lastOrNull()                  // last loaded element, or null
paginator.elementAtOrNull(globalIndex)  // flat index across all cached pages; negative ‚Üí null
```

### Transformation

```kotlin
val all: List<Article> = paginator.flatten()                          // every element, page order
val ids: List<Long>    = paginator.flatMap { it.relatedIds }          // flat-map across pages
val numbers: List<Int> = paginator.mapPages { it.page }               // map per page state

paginator.forEachIndexed { globalIndex, element ->
    /* index is contiguous across page boundaries */
}
```

### Operators & properties

```kotlin
"target" in paginator                  // operator contains
val total = paginator.loadedItemsCount // == paginator.count()
val pages = paginator.loadedPagesCount // == paginator.core.size
```

## `PaginatorCrudExt` ‚Äî element-level CRUD

Receiver is `MutablePaginator<T>`. All bulk operations dispatch a **single** snapshot at the end
(or none, if `silently = true`). All mutations only touch L1 ‚Äî call `flush()` (or wrap in
`transaction { }`) to persist to L2.

### Single-element operations

```kotlin
// Append to the last cached page (returns false if the cache is empty)
paginator.addElement(newItem)

// Insert at an explicit (page, index) coordinate
paginator.addElement(element = newItem, page = 3, index = 0)

// Replace / remove the FIRST match (stops after one hit)
paginator.setElement(element = updatedItem) { it.id == targetId }
val removed: T? = paginator.removeElement { it.id == targetId }
val removed: T? = paginator.removeElement(page = 3) { it.id == targetId }

// Add to the start of the first cached page (symmetric counterpart of addElement)
paginator.prependElement(newItem)
```

### Bulk insert / move / swap

```kotlin
// Swap two elements (same page or across pages)
paginator.swapElements(aPage = 1, aIndex = 0, bPage = 2, bIndex = 4)

// Move an element by (page, index) ‚Äî same semantics as RecyclerView#notifyItemMoved.
// If the source page becomes empty, it is dropped and toPage is shifted down by 1.
paginator.moveElement(fromPage = 1, fromIndex = 0, toPage = 2, toIndex = 0)

// Insert relative to a matching element
paginator.insertBefore(newItem) { it.id == anchorId }
paginator.insertAfter (newItem) { it.id == anchorId }
```

### Bulk update / remove

```kotlin
// Replace every element across all pages
paginator.updateAll { it.copy(read = true) }

// Replace only matching elements; returns how many were touched
val touched: Int = paginator.updateWhere(
    predicate = { it.unread },
    transform = { it.copy(read = true) },
)

// Remove every match; returns the number deleted
val removed: Int = paginator.removeAll { it.id in trashedIds }

// Keep only matching elements (inverse of removeAll); returns the number deleted
val removed: Int = paginator.retainAll { it.published }

// Drop duplicates by key (first occurrence wins); returns the number of dups removed
val dups: Int = paginator.distinctBy { it.id }
```

`removeAll` / `retainAll` / `distinctBy` are implemented on top of `replaceAllElements`, so they
inherit the same rebalance semantics (pages are collapsed when they become empty).

## `PaginatorRefreshExt` ‚Äî full-cache refresh

```kotlin
// Reload every currently cached page in parallel
paginator.refreshAll()

// Suppress UI emissions during loading
paginator.refreshAll(loadingSilently = true, finalSilently = true)
```

For surgical refresh of specific pages (or refresh of dirty pages only), use the built-in
`Paginator.refresh(pages)` method ‚Äî see [Navigation](2.%20navigation.md).

## `PaginatorUiStateExt` ‚Äî high-level UI state

Collapses the raw page snapshot into five UI-friendly variants
(`Idle` / `Loading` / `Empty` / `Error` / `Content`). Full rendering recipe and classification
rules live in [State ‚Üí PaginatorUiState](3.%20state.md#paginatoruistate); this section is a
short API reference.

```kotlin
// Recommended entry point ‚Äî Flow<PaginatorUiState<T>> keyed off the paginator's snapshot.
val uiState: Flow<PaginatorUiState<Article>> = paginator.uiState

// Flow transformer ‚Äî useful if you already have a Flow<List<PageState<T>>> from elsewhere.
val derived: Flow<PaginatorUiState<Article>> =
    paginator.core.snapshot.asUiState(isStarted = { paginator.core.isStarted })

// Pure synchronous mapping ‚Äî convenient in tests and one-off computations.
val single: PaginatorUiState<Article> =
    paginator.core.states.toUiState(isStarted = paginator.core.isStarted)
```

Classification summary:

| Snapshot shape                                            | Result                                      |
|-----------------------------------------------------------|---------------------------------------------|
| `!isStarted` or empty list                                | `Idle`                                      |
| Single `ProgressPage` with empty `data`                   | `Loading(page)`                             |
| Single `EmptyPage` with empty `data`                      | `Empty(page)`                               |
| Single `ErrorPage` with empty `data`                      | `Error(page, exception)`                    |
| Any other shape (including non-success with carried data) | `Content(items, prependState, appendState)` |

`Content.items` flattens the `data` of every visible page; non-success boundary pages at the top
or bottom of the snapshot surface through `prependState` / `appendState` while still contributing
their carried items to `items`. `asUiState` does not apply `distinctUntilChanged` ‚Äî add it
yourself if you want deduplication on the derived structural-equal state.

## `PaginatorPersistentExt` ‚Äî L2 warm-up

`warmUpFromPersistent` eagerly restores L1 from the persistent cache (L2) on cold start,
turning the default lazy `L1 ‚Üí L2 ‚Üí source` fallback into a one-shot hydration. Available
symmetrically on offset `Paginator<T>` and `CursorPaginator<T>`:

```kotlin
import com.jamal_aliev.paginator.extension.warmUpFromPersistent

// Offset Paginator ‚Äî keyed by page: Int
val inserted: Int = paginator.warmUpFromPersistent(
    pageRange = 1..20,   // optional ‚Äî emits a snapshot for the given range
)

// Cursor Paginator ‚Äî keyed by CursorBookmark.self
val insertedCursor: Int = cursorPaginator.warmUpFromPersistent(
    cursorRange = head to tail,   // optional
)
```

Semantics:

| Aspect                    | Behaviour                                            |
|---------------------------|------------------------------------------------------|
| Backend call              | `persistentCache.loadAll()`                          |
| Conflict resolution       | L1 wins ‚Äî pages already in L1 are skipped            |
| Snapshot emission         | None, unless `pageRange` / `cursorRange` is provided |
| CRUD tracker              | Not touched ‚Äî warm-up is not a mutation              |
| `persistentCache == null` | Returns `0` (no-op)                                  |
| Return value              | Number of pages actually inserted into L1            |

**Interaction with eviction.** The warmed-up states go through the normal `cache.setState`
path, so eviction strategies apply on each insert. Pair `warmUpFromPersistent` with an
`LruPagingCache` rather than a `SlidingWindowPagingCache` ‚Äî the latter drops everything that
falls outside the (still-uninitialised) context window.

**Concurrency.** No mutex is acquired. Call from cold start, before any navigation method
fires.

For the full caching picture (auto-persist on source load, `flush()`, transaction rollback,
`hasPendingFlush` / `affectedPages`)
see [Caching ‚Üí Persistent Cache (L2)](6.%20caching.md#persistent-cache-l2).

## Full ViewModel example

A complete ViewModel demonstrating navigation, mutations, and prefetch via the DSL builder:

```kotlin
class PaginatorViewModel : ViewModel() {

    private val _uiState = MutableStateFlow<List<PageState<Article>>>(emptyList())
    val uiState = _uiState.asStateFlow()

    private val paginator: MutablePaginator<Article> = mutablePaginator(capacity = 5) {
        load { page -> LoadResult(repository.loadPage(page)) }
        finalPage = 20
        bookmarks(5, 10, 15)
        recyclingBookmark = true
        logger = PrintPaginatorLogger(minLevel = LogLevel.DEBUG)
    }

    private val prefetch = paginator.prefetchController(
        scope = viewModelScope,
        prefetchDistance = 5,
    )

    init {
        paginator.core.snapshot
            .filter { it.isNotEmpty() }
            .onEach { _uiState.value = it }
            .flowOn(Dispatchers.Main)
            .launchIn(viewModelScope)

        viewModelScope.launch {
            paginator.jump(BookmarkInt(1))
        }
    }

    // 1. Forward / backward pagination (loading indicators wired by the snapshot flow)
    fun loadNextPage()     = viewModelScope.launch { paginator.goNextPage() }
    fun loadPreviousPage() = viewModelScope.launch { paginator.goPreviousPage() }

    // 2. Jump to a user-specified page
    fun jumpToPage(page: Int) = viewModelScope.launch {
        try {
            paginator.jump(BookmarkInt(page))
        } catch (e: FinalPageExceededException) {
            showError("Page exceeds limit")
        }
    }

    // 3. Bookmark navigation
    fun nextBookmark() = viewModelScope.launch { paginator.jumpForward() }
    fun prevBookmark() = viewModelScope.launch { paginator.jumpBack() }

    // 4. Swipe-to-refresh / retry
    fun restart()             = viewModelScope.launch { paginator.restart() }
    fun retryPage(page: Int)  = viewModelScope.launch { paginator.refresh(pages = listOf(page)) }

    // 5. Mutations that should also persist to L2 (if a persistentCache is wired)
    fun markAllRead() = viewModelScope.launch {
        paginator.transaction {
            paginator.updateAll { it.copy(read = true) }
        }
    }

    // 6. Forward scroll position to the prefetch controller
    fun onScroll(firstVisible: Int, lastVisible: Int, total: Int) {
        prefetch.onScroll(firstVisible, lastVisible, total)
    }

    override fun onCleared() {
        paginator.release()
        super.onCleared()
    }
}
```

Rendering in Compose ‚Äî low-level `PageState` approach when you want per-page control:

```kotlin
@Composable
fun PaginatedList(pages: List<PageState<Article>>) {
    LazyColumn {
        pages.forEach { pageState ->
            when {
                pageState.isSuccessState() -> {
                    items(pageState.data.size) { ArticleRow(pageState.data[it]) }
                }
                pageState.isProgressState() -> {
                    items(pageState.data.size) { ArticleRow(pageState.data[it], stale = true) }
                    item { CircularProgressIndicator() }
                }
                pageState.isErrorState() -> {
                    item {
                        ErrorCard(
                            message = pageState.exception.message,
                            onRetry = { retryPage(pageState.page) },
                        )
                    }
                }
                pageState.isEmptyState() -> {
                    item { Text("No data on page ${pageState.page}") }
                }
            }
        }
    }
}
```

Or collapse the same snapshot into a coarse screen state via `paginator.uiState`:

```kotlin
@Composable
fun PaginatedScreen(state: PaginatorUiState<Article>, onRetry: (Int) -> Unit) {
    when (state) {
        PaginatorUiState.Idle       -> Unit
        is PaginatorUiState.Loading -> FullScreenSpinner()
        is PaginatorUiState.Empty   -> EmptyPlaceholder()
        is PaginatorUiState.Error   -> FullScreenError(state.exception) { onRetry(state.page) }
        is PaginatorUiState.Content -> LazyColumn {
            items(state.items) { ArticleRow(it) }

            state.prependState?.let { boundary -> item { BoundaryRow(boundary, onRetry) } }
            state.appendState?.let  { boundary -> item { BoundaryRow(boundary, onRetry) } }
        }
    }
}
```

Pick whichever fits your screen: `snapshot` + `PageState` predicates when you want page-level
granularity (e.g. inline per-page spinners), `uiState` when a full-screen Idle/Loading/Empty/Error
classification is enough.
