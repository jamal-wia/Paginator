## DSL Builders

| Function                                                 | Returns               | Description                                                           |
|----------------------------------------------------------|-----------------------|-----------------------------------------------------------------------|
| `paginator<T>(capacity = DEFAULT_CAPACITY) { … }`        | `Paginator<T>`        | Build a read-only paginator (see [DSL Builder](11.%20dsl-builder.md)) |
| `mutablePaginator<T>(capacity = DEFAULT_CAPACITY) { … }` | `MutablePaginator<T>` | Build a full-featured mutable paginator                               |

Configuration block (shared by both DSLs):

| Property / call           | Type                                          | Description                                                        |
|---------------------------|-----------------------------------------------|--------------------------------------------------------------------|
| `load { fn }`             | `suspend Paginator<T>.(Int) -> LoadResult<T>` | **Required.** Page loader                                          |
| `cache`                   | `PagingCache<T>`                              | L1 cache (default: `DefaultPagingCache()`)                         |
| `persistentCache`         | `PersistentPagingCache<T>?`                   | Optional L2 cache                                                  |
| `logger`                  | `PaginatorLogger?`                            | Logger forwarded to paginator + core                               |
| `finalPage`               | `Int`                                         | Maximum allowed page (default `Int.MAX_VALUE`)                     |
| `recyclingBookmark`       | `Boolean`                                     | Wrap-around bookmark iteration                                     |
| `bookmarks(vararg pages)` | —                                             | Replace bookmark list with `BookmarkInt(p)` for each page          |
| `bookmarks(list)`         | `List<Bookmark>`                              | Replace bookmark list with the supplied `Bookmark` instances       |
| `initializers { … }`      | block                                         | Override `progress` / `success` / `empty` / `error` page factories |

## Paginator Properties

**Direct `Paginator` properties:**

| Property             | Type                                          | Description                                    |
|----------------------|-----------------------------------------------|------------------------------------------------|
| `load`               | `suspend Paginator<T>.(Int) -> LoadResult<T>` | Data source lambda                             |
| `logger`             | `PaginatorLogger?`                            | Logging interface (`null` by default)          |
| `finalPage`          | `Int`                                         | Maximum allowed page (default `Int.MAX_VALUE`) |
| `bookmarks`          | `MutableList<Bookmark>`                       | Bookmark list (default: page 1)                |
| `recyclingBookmark`  | `Boolean`                                     | Wrap-around bookmark iteration                 |
| `lockJump`           | `Boolean`                                     | Lock jump operations                           |
| `lockGoNextPage`     | `Boolean`                                     | Lock forward navigation                        |
| `lockGoPreviousPage` | `Boolean`                                     | Lock backward navigation                       |
| `lockRestart`        | `Boolean`                                     | Lock restart                                   |
| `lockRefresh`        | `Boolean`                                     | Lock refresh                                   |

**Via `paginator.core` (`PagingCore`):**

| Property                   | Type                       | Description                        |
|----------------------------|----------------------------|------------------------------------|
| `core.snapshot`            | `Flow<List<PageState<T>>>` | Visible page states flow           |
| `core.capacity`            | `Int`                      | Expected items per page            |
| `core.isCapacityUnlimited` | `Boolean`                  | `true` if `capacity == 0`          |
| `core.pages`               | `List<Int>`                | All cached page numbers (sorted)   |
| `core.states`              | `List<PageState<T>>`       | All cached page states (sorted)    |
| `core.size`                | `Int`                      | Number of cached pages             |
| `core.dirtyPages`          | `Set<Int>`                 | Snapshot of all dirty page numbers |
| `core.startContextPage`    | `Int`                      | Left boundary of visible context   |
| `core.endContextPage`      | `Int`                      | Right boundary of visible context  |
| `core.isStarted`           | `Boolean`                  | `true` if context pages are set    |

## MutablePaginator Methods (additional)

| Method                                                | Returns         | Description                             |
|-------------------------------------------------------|-----------------|-----------------------------------------|
| `removeState(pageToRemove, silently)`                 | `PageState<T>?` | Remove a page                           |
| `setElement(element, page, index, silently, isDirty)` | `Unit`          | Replace element (optionally mark dirty) |
| `removeElement(page, index, silently, isDirty)`       | `T`             | Remove element (optionally mark dirty)  |
| `addAllElements(elements, targetPage, index, …)`      | `Unit`          | Insert elements (optionally mark dirty) |
| `replaceAllElements(providerElement, …, predicate)`   | `Unit`          | Bulk replace/remove across all pages    |
| `flush()`                                             | `Unit`          | Flush CRUD changes to L2 (suspend)      |

> **Note:** there is no direct `setState` method on `MutablePaginator`. Use
> `paginator.core.setState(state)` or the `paginator += state` operator.

## MutablePaginator Operators

| Operator                 | Description          |
|--------------------------|----------------------|
| `paginator += pageState` | Set page state in L1 |
| `paginator -= page`      | Remove page          |

## Extension Functions

All extensions live in `com.jamal_aliev.paginator.extension`. Receivers are noted in each table.
For full prose & examples see [Extensions](9.%20extensions.md).

### `PageState` extensions (`PageExt`)

| Function                           | Returns   | Description                                             |
|------------------------------------|-----------|---------------------------------------------------------|
| `state.isProgressState()`          | `Boolean` | Smart-casts to `ProgressPage<T>` on `true`              |
| `state.isSuccessState()`           | `Boolean` | Smart-casts to `SuccessPage<T>` (excludes `EmptyPage`)  |
| `state.isEmptyState()`             | `Boolean` | Smart-casts to `EmptyPage<T>`                           |
| `state.isErrorState()`             | `Boolean` | Smart-casts to `ErrorPage<T>`                           |
| `state.isRealProgressState(clazz)` | `Boolean` | True if the state is a specific `ProgressPage` subclass |
| `state.isRealSuccessState(clazz)`  | `Boolean` | True if the state is a specific `SuccessPage` subclass  |
| `state.isRealEmptyState(clazz)`    | `Boolean` | True if the state is a specific `EmptyPage` subclass    |
| `state.isRealErrorState(clazz)`    | `Boolean` | True if the state is a specific `ErrorPage` subclass    |
| `a near b`                         | `Boolean` | Page distance is 0 or 1                                 |
| `a far  b`                         | `Boolean` | Page distance is greater than 1                         |
| `a gap  b`                         | `Int`     | `abs(a.page - b.page)`                                  |

| `setElement(element, silently, predicate)`                    | `Unit`    | Replace the FIRST element matching `predicate`                               |

| `swapElements(aPage, aIndex, bPage, bIndex, silently)`        | `Unit`    | Swap two elements                                                            |

## `PaginatorUiState` variants

`PaginatorUiState<T>` is a sealed interface living in `com.jamal_aliev.paginator.page`. The mapper
rules are documented in [State → PaginatorUiState](3.%20state.md#paginatoruistate); this table is
a quick reference for the variants themselves.

| Variant                                                      | Fields                                                                        | Emitted when                                                                        |
|--------------------------------------------------------------|-------------------------------------------------------------------------------|-------------------------------------------------------------------------------------|
| `PaginatorUiState.Idle` (object)                             | —                                                                             | `!isStarted` or the snapshot is empty (fresh paginator, after `release()`).         |
| `PaginatorUiState.Loading(page)`                             | `page: Int`                                                                   | Snapshot is exactly one `ProgressPage` with `data.isEmpty()`.                       |
| `PaginatorUiState.Empty(page)`                               | `page: Int`                                                                   | Snapshot is exactly one `EmptyPage` with `data.isEmpty()`.                          |
| `PaginatorUiState.Error(page, exception)`                    | `page: Int`, `exception: Exception`                                           | Snapshot is exactly one `ErrorPage` with `data.isEmpty()`.                          |
| `PaginatorUiState.Content(items, prependState, appendState)` | `items: List<T>`, `prependState: PageState<T>?`, `appendState: PageState<T>?` | Any other snapshot shape — covers paginated content and reloads with carried items. |

`Content.items` is the flattened `data` of every visible page, in snapshot order.
`Content.prependState` / `Content.appendState` are the first / last visible pages when they are
**not** `SuccessPage` (e.g. `ProgressPage` loading the previous / next page, or the corresponding
`ErrorPage`); otherwise `null`. Non-success boundary states still contribute their carried items
to `items` so the last known content stays visible while a reload is in flight or has failed.
